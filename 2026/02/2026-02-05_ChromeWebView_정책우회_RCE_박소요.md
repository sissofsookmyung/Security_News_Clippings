## [2026-02-05]_[Chrome_WebView_정책우회_RCE]__[박소요].md

## 1. 기본 정보

- **제목**: 크롬 웹뷰(WebView) 정책 우회 취약점으로 승인되지 않은 데이터 접근 및 RCE 위험
- **발생 일자**: 명확히 특정되지 않음 (내부 선행 발견)
- **보도 일자**: 2026-01-08

## 2. 사건 / 이슈 요약 (Fact)

> 기사에서 확인된 객관적 사실만 정리

- **어떤 사건인지**:  
  구글 크롬 브라우저의 웹뷰(WebView) 태그 구성 요소에서 보안 정책 강제 집행이 불충분하게 이루어지는 고위험 취약점(CVE-2026-0628)이 발견됨.  
  해당 취약점은 웹뷰 내부에서 보안 제한을 우회할 수 있는 정책 집행 미비로 인해 발생함.

- **피해 대상(조직/산업군/국가)**:  
  크롬 웹뷰를 사용하는 모든 사용자 및 애플리케이션 환경  
  (윈도우, 맥, 리눅스, 안드로이드 포함)

- **피해 규모(확인된 경우)**:  
  실제 피해 사례는 기사 기준으로 공개되지 않음

- **사건 상태(진행 중 / 종료 / 조사 중)**:  
  구글에서 취약점 확인 후 긴급 보안 업데이트 배포 완료

    - 웹뷰(WebView) 태그의 보안 정책 우회

    - CVE-2026-0628
- **악성코드/도구**:

    - 보안 제어 우회를 통한 승인되지 않은 리소스 접근 및 코드 실행 가능성

## 6. 이슈의 의미 (분석)

> 왜 중요한지, 기존 사례와의 차이점

- **기술적 관점**:
    - 브라우저 메인 기능이 아닌 하부 구성 요소(WebView)에서 발생한 취약점
    - 취약점은 렌더러나 네트워크 레이어가 아닌 WebView/GuestView 같은 보조 컴포넌트의 정책 로직에서 발생했지만 결과적으로 브라우저 전체 보안 경계를 우회할 수 있는 수준의 영향 (C/I/A High)를 가짐 
    - 악성 확장 + WebView 정책 우회를 결합해 privileged chrome:// 페이지에 코드 주입이 가능하다는 점에서, 전형적인 XSS/CSRF보다 한 단계 높은 권한 탈취가 가능
    - 앱 내 웹 렌더링 구조 특성상 보안 경계가 불명확함

- **보안 트렌드 관점**:
    - 웹 기술 복잡성 (복잡한 샌드박스/정책 계층을 도입) 증가로 정책 집행 미비 취약점 지속 발생
    - 단순 브라우저 취약점 x , WebView를 임베딩한 애플리케이션 생태계 전체에 위험을 전이시키는 형태
    - 책임 있는 공개(RDP)와 패치 선행의 중요성 재확인

- **조직/산업에 주는 시사점**:
    - 사용자 단말 보안 수준이 곧 조직 보안 수준으로 직결
    - 웹뷰 기반 내부 업무 앱 보안 점검 필요성 증가
