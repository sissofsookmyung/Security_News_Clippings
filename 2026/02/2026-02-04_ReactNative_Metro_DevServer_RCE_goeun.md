## 2. 사건 / 이슈 요약 (Fact)

> 기사에서 확인된 객관적 사실만 정리

- 어떤 사건인지:
  - 공격자들이 React Native Community CLI가 띄우는 Metro 개발 서버에 존재하는 **OS Command Injection 기반 RCE 취약점(CVE-2025-11953, 별칭 Metro4Shell)**을 실제로 악용한 정황이 관측됨 
- 피해 대상(조직/산업군/국가):
  - 인터넷에 노출된 Metro 개발 서버(개발/테스트 환경 포함) 운영 호스트가 공격 표면이 됨
- 피해 규모(확인된 경우):
  - “수천 개”의 인터넷 접근 가능 인스턴스가 위험할 수 있다고 언급됨(노출 규모 관점)
- 사건 상태(진행 중 / 종료 / 조사 중):
  - 2025-12-21 이후 2026년 1월에도 반복 관측되어 **지속 악용(운영적 사용)** 정황

## 4. 공격 방식 및 기술적 요소

- **공격 벡터**:
  - **취약점 악용**: 인터넷에 노출된 Metro 개발 서버에 원격 요청을 보내 **인증 없이 명령 실행**
- **사용된 취약점(CVE)**:
  - **CVE-2025-11953 (Metro4Shell)** / CWE-78(OS Command Injection), CVSS 9.8
- **악성코드/도구**:
  - Base64 인코딩된 **PowerShell 스크립트**
  - (후속) **Rust 기반 바이너리**(안티분석 체크 포함) 
- **특징적인 기술**:
  - Microsoft Defender Antivirus 제외(Exclusion) 등록(작업 디렉터리/Temp 경로 등) 
  - 공격자 제어 호스트/포트로 **TCP 연결(C2 유사)** 후, 추가 페이로드 다운로드·실행

## 5. 대응 및 후속 조치

- **피해 조직의 대응**:
  - (원칙) 노출된 개발 서버 즉시 차단/내부망 격리, 해당 호스트의 무결성 점검 및 필요 시 재이미징(침해 가정) — 기사/관측 흐름상 권고되는 대응 
  - VulnCheck가 **야생 악용(EITW) 관측** 및 타임라인/행위 상세 공개
- **권고 사항**:
  - Metro/React Native Community CLI 관련 패키지 **패치 적용**(CVE-2025-11953 수정 버전으로 업데이트) 
  - Metro 개발 서버가 기본적으로 외부 인터페이스에 바인딩될 수 있어 **인터넷 노출 금지(방화벽/바인딩 설정 점검)**
  - Defender 제외 등록/의심 PowerShell 실행/비정상 외부 접속(특정 IP/포트) 로그 점검

## 6. 이슈의 의미 (분석)

> 왜 중요한지, 기존 사례와의 차이점

- 기술적 관점:
  - “개발 서버”로 여겨지는 Metro가 **인증 없는 RCE** 공격 표면이 되었고, 단순 요청으로 OS 명령 실행이 가능하다는 점이 핵심 
- 보안 트렌드 관점:
  - **개발 인프라(Dev/CI/Tooling) → 실전 침해 경로**로 전환되는 패턴을 재확인(노출되는 순간 운영 인프라가 됨) 
- 조직/산업에 주는 시사점:
  - “개발 편의” 설정(외부 바인딩/노출)이 곧바로 침해로 이어질 수 있어, 개발 환경도 **자산관리/노출관리/패치 프로세스**에 포함해야 함

## 8. 정리 메모

- 한 줄 요약:
  -Metro 개발 서버 안에 원래 “개발 편의용 OS 명령 실행 기능”이 있었는데, 
  - 그 부분에 취약점이 있어서 공격자가 
  - 1. 네트워크로 Metro 서버에 접근하고
  - 2. OS 명령을 원격으로 실행한 뒤
  - 3. 로컬 개발자 PC에 악성코드를 심고
  - 4. C2 서버와 연결해서
  - 5. 그 PC를 계속 원격 조종할 수 있게 만든 것
